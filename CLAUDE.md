# 체납대장 워크스테이션 - 작업 규칙

## 브랜치
- **main 브랜치만 사용**. 별도 feature 브랜치 생성 금지.
- 변경 후 항상 main에 바로 커밋 & 푸시.

## 버전 시스템 (X.X.X.X)

| 자리 | 의미 | 예시 |
|------|------|------|
| 첫째 | 시스템 전면 교체 (아키텍처/플랫폼 변경) | 1.x.x.x → 2.x.x.x |
| 둘째 | 대규모 기능 업데이트 | x.1.x.x → x.2.x.x |
| 셋째 | 소규모 기능 추가 | x.x.1.x → x.x.2.x |
| 넷째 | 버그 수정 / 오류 수정 | x.x.x.1 → x.x.x.2 |

- **업데이트(3·2번째 자리)** = 기능 추가 위주
- **수정(4번째 자리)** = 기능 변경 없는 버그픽스

## 파일명
- 현재 파일: `체납대장_워크스테이션_v2.1.3.2.html`
- 파일명의 버전은 **최초 배포 버전** 기준이므로 변경하지 않음.
- 버전은 파일 내부 주석 헤더와 `<title>` 태그에서 관리.
